飞狐 API接入指南
欢迎使用 飞狐 API。所有接口支持 HTTP GET/POST，返回统一为 JSON，便于快速集成。

API 系统正常运行
上次更新：2025-11-11
基础地址
http://api.feihu2026.com
认证方式
调用登录接口获取 token，GET/POST 通过 token 字段传递
请求方式
HTTP GET / POST
返回格式
application/json · UTF-8
通用返回结构
所有接口返回以下结构，仅 data 字段因业务而异。

{
  "code": 200,
  "msg": "OK",
  "data": { ... }
}
GET / POST
登录获取 Token
使用 api_key 与 api_pass 换取访问令牌。建议在程序启动时调用一次并缓存(一直有效)，避免频繁登录。

请求 URL
/api/sms/login
复制
请求参数
字段	必选	类型	说明
user	是	string	api_key
pass	是	string	api_pass
data 字段
字段	类型	说明
token	string	后续接口使用的访问令牌
响应示例
复制响应
{
  "code": 200,
  "msg": "OK",
  "data": {
    "token": "eyJhbGciOiJI..."
  }
}

获取手机号
获取指定项目的手机号，可按运营商、地区、号段等维度筛选。GET 请求通过 query 传参，POST 使用表单字段。

province	否	string	省份，需传枚举数字
cardType	否	string	1=实卡 / 2=虚卡，空为全部
include	否	string	号段包含（多个用|分隔）
exclude	否	string	号段排除（多个用|分隔）
author	否	string	号码作者（可选）
data 字段
字段	类型	说明
pid	int64	项目 ID
phone	string	分配的手机号
province	string	省份
city	string	城市
isp	string	运营商
order_id	string	订单号
last_event_time	string	号码最近更新时间

GET / POST
获取短信内容
根据订单号获取最新短信内容，轮询时建议间隔 ≥ 10 秒。

请求 URL
/api/sms/getMessage
复制
请求参数
字段	必选	类型	说明
token	是	string	登录返回的 token
order_id	是	string	获取号码时返回的订单号
data 字段
字段	类型	说明
order_id	string	订单号
phone	string	手机号
message	string	完整短信内容
code	string	识别出的验证码（若可提取）
sms_received_at	string	短信接收时间
status	int	订单状态，见下方 status 取值说明
status 取值
1
等待短信到达
已下单，短信还未返回
3
已完成
成功接收到短信内容
4
失败
请检查余额是否不足或其他异常
5
已释放/不存在
订单已释放或无效
6
超时
超过有效期仍未收到短信
响应示例
复制响应
{
  "code": 200,
  "msg": "OK",
  "data": {
    "order_id": "xxxxxxxx",
    "phone": "13800138000",
    "message": "【xxxxx】验证码 4829，请勿泄露。",
    "code": "4829",
    "sms_received_at": "2024-01-01 12:00:05",
    "status": 3
  }
}
若暂未收到短信，status=1 时表示未收到，請保持輪詢。收到短信后 status 会变为 3，表示已完成。
